# 32｜局限与起点：让 Hify 成为你真正的项目经验

你好，我是Robert。

在讲这节课的重点之前，我想先和你讨论一个问题：**怎么写出一个好的提示词？**这个问题本身没有通用答案。

你能问出“用Redis还是MySQL做上下文缓存”，背后是你对两种存储特性的理解。你能问出“可观测性怎么做”，背后是你做过生产系统、踩过“出了问题什么都看不到”的坑。你能问出“同步导出还是异步导出”，背后是你对并发场景和用户体验的判断。

**提示词的质量，是你认知深度的外化。**课程中我们学习了咨询模式、SDD、任务拆解等等，但真正决定你问出什么层次问题的，是经验积累，也是这门课想交付给你的另一条主线。

**这门课可以成为你开启Vibe Coding的主线，你学习的主干。只要你旁征博引，保持好奇心，去不断展开，我相信这个项目就是你真正的经验**。

## Vibe Coding的本质：是驾驭，不是依赖

Vibe Coding听起来是跟着感觉走，但真正能跑通的人，都有积累在背后撑着。

没有积累的Vibe Coding是什么样的？Claude Code给了五个方案，你不知道选哪个；Claude Code生成了代码，你看不出哪里有隐患；Claude Code问你“用Redis还是MySQL”，你没有判断依据，只能说“你觉得哪个好就哪个”。最后的结果是：AI在驾驭你，不是你在驾驭AI。

有积累的Vibe Coding是什么样的？你知道这个场景的约束条件，所以你能给出有意义的限定；你见过类似的问题，所以你知道Claude Code给的方案哪里可能踩坑；你对这个领域有基本认知，所以你能在Claude Code偏离方向时把它拉回来。

**积累不是AI能替代的东西**。计算机基础、项目经验、踩过的坑、做过的技术决策，这些是你真正的资产。AI工具是放大器，放大的是你已有的东西。你有判断力，好工具放大判断力；你没有积累，好工具放大的是错误。

这就是为什么Vibe Coding的本质是驾驭，你需要足够的积累，才能真正坐在驾驶位上。

## 收获：这门课给了你什么，没给你什么

**给了你的**：

一个完整的项目经验。从产品定义、架构设计、核心开发、测试、部署到可观测性，Hify走完了一个AI应用的全链路。这本身就是积累，一个真实的、有上下文的项目经验，比看10篇教程有价值。

一套可复用的工作方式。咨询模式、SDD、任务拆解、Skills、CLAUDE.md，这些方法论不绑定Hify，换任何项目都能用。

一个认知建立的方法。15讲的领域四问法，让你在遇到陌生技术时知道怎么快速建立判断依据。这个方法比任何具体知识点都更有价值，可以迁移到任何领域。

**没给你的**：

计算机基础的厚度。数据结构、操作系统、网络协议，这些是问出好问题的底层，Hify用到了它们，但这门课不是专门教这些的。

垂直领域的深度。RAG往深走是重排序、混合检索、RAG评估体系；Agent往深走是多步规划、多Agent协作；分布式系统往深走是一致性协议、故障恢复。每个方向都是独立的大领域。

大规模系统的经验。Hify面向几十人，不是百万用户。高并发、极致性能优化、超大规模工程化，这些需要另外的项目经验积累。

说清楚这个边界，是为了让你知道接下来该往哪里走。

## 积累：把Hify当作主干，向外展开

这门课的真正价值不只是学会了一套工具和方法，而是你有了一个完整的、有上下文的项目主干。

主干的价值是：**它给了你一个锚点，让你知道每个知识点“长在哪里”**。

你学数据库事务，知道它解决的是Hify里Redis read-modify-write那个并发问题。你学HTTP协议，知道它和SSE流式响应直接相关。你学K8s健康检查，知道它是readinessProbe那几行配置背后的逻辑。你学向量数据库，知道它是RAG管线里分块向量化那个环节用到的东西。

有了主干，零散的知识点有地方挂，学了能消化，不会读完就忘。没有主干，学了也是浮的，停留在“看懂了”而不是“理解了”。

**旁征博引，保持好奇心**。做Hify的过程中，你一定遇到了很多没完全想透的东西，为什么SSE比WebSocket更适合这个场景？pgvector的索引是怎么工作的？K8s的readiness和liveness有什么本质区别？每一个没想透的地方，都是可以展开的线索。

沿着这些线索往深走。不需要立刻把每个方向都走透，但每次遇到一个疑问，养成“往下看一层”的习惯。今天多理解了一个知识点，下次问Claude Code的问题质量就高一点，Claude Code给你的输出质量就高一点，这是一个正向循环，而且会加速。

用Claude Code加速这个过程。让它帮你解释概念，帮你做最小Demo验证理解，帮你出题测试完成学习闭环。Claude Code是很好的学习伙伴，不只是写代码的工具。

## 复盘：真正带走了什么

几个问题，停下来对照自己想一想。

**1. 哪些决策你真的理解了，哪些只是看懂了？**

能说清楚“为什么这么做，不那么做”的地方，是真正理解的地方。说不清楚的地方，知识还没落地。

**2. 哪些环节你最依赖Claude Code，几乎没有自己的判断？**

那些地方就是知识盲区。不是坏事，是方向，接下来值得在那里多投入。

**3. 有没有哪个技术选型，当时完全跟着Claude Code走，现在也说不清楚为什么？**

这样的选型值得回头想一想。不是要否定它，而是把“接受了Claude Code的建议”变成“理解了为什么这是对的”。理解了之后，下次遇到类似场景你能自己判断。

**4. 做完Hify，你的知识盲区清单是什么？**

列出来。这个清单比这门课的任何内容都更有价值，它是专属于你的学习路线图，是接下来系统补基础的方向。

## 总结

好的提示词是认知深度的外化。这门课教不全，但给了你一个很好的起点：**一个完整的项目主干、一套工作方式、一个比较清晰的“我还缺什么”的认知地图。**

计算机基础要补，项目经验要积累，垂直领域要深入，这些不会因为有了AI工具就不重要了，反而更重要。因为你的积累决定了你能问出什么层次的问题，问题层次决定了AI能帮你多少。

只要你旁征博引，保持好奇心，沿着Hify的每一根线索往外展开，这个项目就不只是一门课的练习，它是你真正的项目经验，你的积累，你能问出更好问题的基础。

在Claude Code的帮助下，成长可以更快。但成长的方向和深度，永远是你自己决定的！
